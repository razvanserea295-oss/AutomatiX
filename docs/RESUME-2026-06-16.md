## A. UI ↔ BACKEND SEPARATION (rebuild UI from 0)

**The backend is already a standalone API server — it does NOT import `src/`.** The UI is a pure SPA that calls the API. So a brand-new UI just needs to speak the same HTTP contract.

### The contract (everything the UI uses)
- **Every call:** `POST {serverUrl}/api/cmd/<command>` with JSON body = args, header `Authorization: Bearer <token>` (all except `login`). Response = JSON (command's return value); on error, non-2xx with `{ code, message }`. Reference client: `src/api/commands.ts` (`apiCommand()` — copy this into the new UI).
- **Auth:** `POST /api/cmd/login { username, password }` → `{ token, … , must_change_password }`. Store the token, send it as Bearer. `change_password` clears the forced-change flag. Seeded admin: `admin` / `1234` (forced password change on first login; policy ≥12 chars upper+lower+digit+symbol).
- **Tenant/health:** `GET /api/health` → `{ status, version, businessType: 'manufacturing'|'restaurant', tenantName }`. Drives which nav/pages to show.
- **Request-wrapped commands:** a set (see `REQUEST_WRAPPED_COMMANDS` in `src/api/commands.ts`) expect `{ request: {...} }`, but handlers read `args.request || args`, so flat args also work.
- **284 commands total.** List them all: `grep -rhoE "ipcRegister\('[a-z_]+'" electron/ | sort -u`. Each command's logic + **data shapes (TS interfaces)** live in `electron/services/*.ts`; HTTP wiring in `electron/ipc/*.ts`. The existing `src/store/*.ts` show which commands each feature uses + the client-side types — best reference when rebuilding a feature.

### Run the backend standalone
- `npm run server` (= `tsc -p tsconfig.server.json && node dist-server/server/index.js`) → API on `http://localhost:3500` (`PROMIX_PORT` to change). It also statically serves `dist/` (the built SPA) and falls back to `index.html` for deep links.
- DB = AES-GCM-encrypted SQLite (`data/promix.db`, key `data/.dbkey`); `migrations/NNN_*.sql` auto-apply on boot. Multi-tenant: `PROMIX_DATA_DIR` + `scripts/serve-tenant.cjs <slug> <port> "Name"` (ZET runs on :3501 via `serve-tenant.cjs zet-burgers 3501`).
- CORS for a separate dev UI: set `PROMIX_ALLOWED_ORIGINS=http://localhost:5173` (or use a dev proxy — `vite.config.ts` already proxies `/api` and `/t` → `:3500`).

### To rebuild the UI from 0
1. New frontend (e.g. fresh Vite React app) in a new folder OR replace `src/`.
2. Copy `src/api/commands.ts` (+ `src/config/server.ts`, `src/config/localStorage.ts`) as the API client, OR write a minimal `fetch` wrapper to `POST /api/cmd/<cmd>` with Bearer.
3. Build login → token → call commands. Use `electron/services/*.ts` for response shapes and the command list to know what's available.
4. Point it at the running backend (`:3500` / `:3501`). Backend needs ZERO changes.
- Keep-if-useful from the old UI: `src/store/*.ts` (Zustand data layer per feature — already thin wrappers over `apiCommand`).
